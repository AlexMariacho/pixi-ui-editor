# Итерация 05 — инструменты viewport и окно Assets

## Зачем нужна эта итерация

Редактор умеет рендерить реальные assets, но работа внутри viewport остаётся неудобной: панорамирование доступно только средней кнопкой мыши, размер объекта можно поменять только цифрами в Inspector, а панель Assets зажата внизу Hierarchy и не масштабируется на большее число ассетов. Эта итерация добавляет Unity-подобную вертикальную панель инструментов с тремя режимами работы во viewport (Pan, Select, Resize с ручками на выбранном объекте) и выносит Assets в отдельное плавающее окно с переключаемым режимом отображения (список / сетка превью).

## Главное правило итерации

**Каждая задача заканчивается результатом, который можно открыть в браузере и проверить глазами.** Раздел «Визуальная проверка» обязателен к выполнению и к описанию в отчёте исполнителя.

## Зафиксированные решения итерации

Приняты владельцем проекта; исполнители не выбирают альтернативы и не «улучшают» их:

- **Схема, `packages/schema` и `packages/runtime-pixi` не меняются.** Вся итерация — только `apps/editor-web`.
- Активный инструмент — **transient-состояние store** (`activeTool: "pan" | "select" | "resize"`), как `activeProfile`: не входит в документ и не переживает перезагрузку. Default — `select`.
- Горячие клавиши инструментов — **Q / W / E** (Pan / Select / Resize), игнорируются, когда фокус в `input`, `textarea`, `select` или contentEditable-элементе.
- Панорамирование средней кнопкой и zoom колесом работают **во всех режимах** — существующее поведение не трогаем.
- В режиме Select сохраняется текущее поведение: клик выбирает node, drag тела перемещает его, клик по пустому месту снимает selection. Отдельного инструмента Move нет.
- Resize редактирует **`transform.width` / `transform.height`** (плюс `x`/`y` при перетягивании левых/верхних ручек), а не `scaleX`/`scaleY`. Все изменения идут через существующий `updateNodeProfileTransform` (он уже per-profile), значения округляются до 2 знаков, минимальный размер — 1.
- Поворот (rotation ≠ 0) resize-ручками **не поддерживается**: ручки строятся по bounds, поведение на повёрнутых nodes может быть неточным — осознанное упрощение прототипа.
- UI-преференсы редактора (открытость и позиция окна Assets, режим отображения) хранятся в **отдельном ключе localStorage `pixi-ui-editor:ui-prefs`** и никогда не попадают в project document.
- Все изменения документа — только через операции store с `commitCandidate`; сцена по-прежнему полностью пересоздаётся при любом изменении документа.
- Команды запускаются из PowerShell; `pnpm build` обязан оставаться зелёным после каждой задачи.

## Демонстрируемый результат

`pnpm --filter @pixi-ui-editor/editor-web dev` открывает редактор, в котором:

1. Кнопка Assets в верхнем toolbar открывает отдельное плавающее окно Assets (перетаскивается за заголовок, закрывается крестиком); из панели Hierarchy раздел Assets исчез, весь функционал (Upload / Replace / Delete / usage count) сохранён;
2. в заголовке окна Assets переключается режим отображения List / Grid; Grid показывает крупные превью плитками; выбранный режим и позиция окна переживают перезагрузку страницы;
3. у левого края редактора — вертикальная панель инструментов Pan (Q) / Select (W) / Resize (E); в Pan левый drag двигает камеру и не трогает selection, в Select всё работает как раньше;
4. в Resize у выбранного node по углам и серединам сторон появляются крупные точки; перетягивание точек меняет размер объекта на canvas и значения в Inspector, независимо для Horizontal и Vertical.

## Общий Definition of Done

- `pnpm build`, `pnpm typecheck`, `pnpm test` завершаются с кодом `0` (в этом порядке);
- все четыре пункта демонстрируемого результата воспроизводятся вручную;
- `packages/schema` и `packages/runtime-pixi` не изменены;
- невалидный документ никогда не попадает в store (существующие тесты store зелёные);
- ADR 0005 записан, секция «Текущее состояние» в `agents.md` обновлена.

## Не входит в итерацию

- rubber-band / marquee-выделение рамкой, множественное выделение;
- инструменты Rotate и Scale, модификаторы (Shift — пропорции, Alt — от центра), snapping;
- корректный resize повёрнутых nodes;
- drag-and-drop asset из окна Assets на canvas, переименование assets, папки, поиск;
- изменение размера самого окна Assets, docking, несколько окон;
- undo/redo, prefabs, Spine, exporter.

Если для выполнения задачи приходится реализовывать что-либо из этого списка, остановитесь: scope выбран неверно.
