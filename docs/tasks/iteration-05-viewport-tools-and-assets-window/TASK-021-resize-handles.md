# TASK-021 — Resize-ручки на выбранном объекте, ADR 0005

## Зависимость

TASK-020 завершена. Сначала выполните её проверки, включая визуальную.

## Цель

В режиме Resize у выбранного node появляются крупные точки по углам и серединам сторон. Перетягивание точки меняет размер объекта прямо на canvas (значения синхронно меняются в Inspector), независимо для Horizontal и Vertical ориентаций.

## Зафиксированные решения

- Ручки показываются только при `activeTool === "resize"` и только у выбранного node; в остальном Resize ведёт себя как Select (клик выбирает, drag тела перемещает).
- **8 ручек**: 4 угла + 4 середины сторон. Позиции — по `getBounds()` node view (как selection-рамка). Отрисовка — квадраты ~10×10 px в **screen space** (постоянный размер независимо от zoom), заливка белая, обводка `SELECTION_COLOR`. Курсоры — соответствующие направлениям (`nwse-resize`, `ns-resize` и т.д.).
- Ручки живут в отдельном `Container` поверх существующего selection-overlay; у этого контейнера `eventMode` интерактивный (существующий overlay остаётся `eventMode: "none"`). Перерисовка — вместе с selection-рамкой (`redrawSelectionRef` или аналогичный механизм).
- Математика drag — в **parent-local координатах** node (как в `moveDraggedNode`): противоположный край/угол зафиксирован; угловые ручки меняют обе оси, ручки середин сторон — одну. При перетягивании левых/верхних ручек обновляются также `x`/`y`.
- Запись — только через существующий `updateNodeProfileTransform` (он уже профильно-зависимый): патч из полей `x`, `y`, `width`, `height`. `scaleX`/`scaleY` не трогаем. Значения округляются до 2 знаков, минимальные `width`/`height` — 1 (ручка «упирается», отрицательных размеров и переворота нет).
- Сцена пересобирается на каждый commit, поэтому во время drag node view ищите заново по id (тот же приём, что в `moveDraggedNode`).
- Rotation ≠ 0 не поддерживается: ручки строятся по bounds, на повёрнутых nodes поведение может быть неточным — осознанное упрощение (у sample-nodes rotation = 0).
- **ADR 0005** (`docs/adr/0005-viewport-tools-and-ui-prefs.md`) фиксирует решения итерации: transient tool-режимы в store; Resize редактирует `width`/`height`, а не scale; UI-преференсы — отдельный localStorage-ключ вне документа. Формат — по образцу ADR 0004.

## Визуальная проверка

1. Выберите Logo, нажмите E → вокруг Logo 8 крупных точек поверх selection-рамки.
2. Потяните правый нижний угол → Logo растёт за курсором; Width/Height в Inspector меняются синхронно; левый верхний угол остаётся на месте.
3. Потяните левую среднюю ручку влево → ширина растёт, правый край на месте (x уменьшается).
4. Сожмите node до предела → размер останавливается на 1, переворота нет.
5. Переключитесь в Vertical → у Logo там прежний размер; измените его ручками → вернитесь в Horizontal — там размер из шага 2 (профили независимы).
6. Позумьте колесом → точки остаются одного экранного размера и на углах node.
7. Нажмите W → точки исчезли, drag снова просто перемещает node.
8. Перезагрузите страницу → изменённые размеры сохранились.
