# TASK-020 — панель инструментов и режимы Pan / Select

## Зависимость

TASK-019 завершена. Сначала выполните её проверки, включая визуальную.

## Цель

У левого края редактора появляется вертикальная панель инструментов (как в Unity/Figma) с тремя кнопками: **Pan (Q)**, **Select (W)**, **Resize (E)**. В этой задаче реализуются режимы Pan и Select; кнопка Resize уже есть и переключает режим, но во viewport Resize пока ведёт себя как Select (ручки — TASK-021).

## Зафиксированные решения

- В `useEditorStore` добавляется transient-состояние (не персистится, не входит в документ, `commitCandidate` не нужен):

```ts
activeTool: "pan" | "select" | "resize"; // default: "select"
setActiveTool(tool: "pan" | "select" | "resize"): void;
```

- Панель — новая самая левая колонка grid-раскладки `.editor-shell` (левее Hierarchy), ширина ~48px, на всю высоту под toolbar. Кнопки ~40×40px, иконки — символы или inline-SVG на усмотрение исполнителя, у каждой `title` вида `Pan (Q)`; активная кнопка подсвечена.
- Горячие клавиши: `keydown` на `window` — `Q` → pan, `W` → select, `E` → resize. Событие игнорируется, если фокус в `input`, `textarea`, `select` или contentEditable, а также при зажатых Ctrl/Alt/Meta.
- Поведение viewport по режимам (все правки — в `SceneCanvas` в `App.tsx`):
  - **Pan**: левый drag по canvas панорамирует камеру — переиспользуйте существующую pan-логику средней кнопки (расширьте условие в `onPointerDown`: средняя кнопка всегда, левая — только при `activeTool === "pan"`; текущий режим читайте через `useEditorStore.getState()`). Клики по nodes в pan-режиме не выбирают и не двигают их (early return в pointerdown-обработчике node), клик по пустому месту не сбрасывает selection. Курсор над canvas — `grab`, во время drag — `grabbing`.
  - **Select**: текущее поведение без изменений — клик выбирает node, drag тела перемещает, клик по пустому месту снимает selection.
  - **Resize**: в этой задаче идентичен Select.
- Панорамирование средней кнопкой и zoom колесом работают во всех режимах.
- Смена инструмента не сбрасывает selection.

## Что создать

- `activeTool` / `setActiveTool` в `store.ts`.
- Компонент панели инструментов (внутри `App.tsx` или отдельным файлом — на усмотрение исполнителя), hotkeys, правки поведения в `SceneCanvas`, стили.

Новых unit-тестов не нужно (`setActiveTool` — тривиальный setter, тестировать нечего).

## Визуальная проверка

1. Откройте редактор → слева вертикальная панель, активен Select (подсвечен).
2. В Select: клик по Logo выбирает его, drag перемещает, клик по пустому месту снимает selection — как раньше.
3. Нажмите Q (или кнопку Pan) → курсор над canvas `grab`; левый drag двигает камеру; выбранный до этого node остаётся выбранным; клики по nodes ничего не выбирают.
4. В Pan: колесо зумит, средняя кнопка тоже панорамирует.
5. Нажмите W → Select снова работает. Нажмите E → кнопка Resize подсвечена, клик/drag ведут себя как в Select.
6. Поставьте фокус в поле имени node в Inspector и нажмите Q → инструмент не переключился, буква напечаталась.

## Критерии приёмки

- `activeTool` не попадает ни в документ, ни в localStorage.
- Все шесть шагов визуальной проверки воспроизводятся.
- Обязательные команды зелёные.
