# TASK-007 — inspector свойств с live-обновлением сцены

## Зависимость

TASK-006 завершена. Сначала выполните её проверки, включая визуальную.

## Цель

Свойства выбранного node редактируются в правой панели без правки JSON; canvas обновляется немедленно.

## Зафиксированные решения

- В store добавляется одна операция изменения документа:

```ts
updateNode(nodeId: string, patch: Partial<Pick<UINode, "name" | "visible" | "transform">> & { text?: string }): void;
```

Реализация: `structuredClone` документа (или immutable-обновление вручную), применение patch, затем `validateProjectDocument`; невалидный результат **не** записывается в store (изменение отбрасывается, `console.warn` с issues). Это защита от порчи документа, а не UX-фича.

- Редактируется **базовый** `transform`; `layoutOverrides` в этой задаче не трогаются.
- Числовые поля — `<input type="number">`; применение по `onChange`; нечисловое/пустое значение не применяется (input может временно показывать невалидное состояние — это допустимо).
- Canvas реагирует на изменение документа полным rebuild (решение итерации), выделение и рамка сохраняются.

## Что создать

Компонент `Inspector` в правой панели. Для выбранного node отображает и редактирует:

- `name` (text input) и `visible` (checkbox);
- секция Transform: `x`, `y`, `width`, `height`, `scaleX`, `scaleY`, `rotation` — семь number inputs (подписи слева, шаг 1; для scale шаг 0.1);
- для `text`-node дополнительно поле `text` (text input);
- read-only строки: `id` и `type` (id — мелким шрифтом, копировать/редактировать не нужно).

Если ничего не выбрано — прежний текст `Выберите node`.

В тесты добавьте один unit-тест store (vitest, без DOM): `updateNode` меняет `x` и имя; patch, делающий документ невалидным (например, `width: 0`), отбрасывается и store не изменяется.

Не реализуйте undo, drag-по-label для чисел, выражения в полях, редактирование `binding`, `assetId` и layout-профилей.


## Визуальная проверка

1. Выберите image-node, меняйте `x` стрелками input → прямоугольник двигается синхронно.
2. Поменяйте `rotation` на 0.5 → прямоугольник повернулся, рамка выделения следует за ним.
3. Снимите `visible` → node исчез с canvas, остался в дереве.
4. У text-node измените `text` → текст на canvas изменился; измените `name` → строка в дереве переименовалась.
5. Введите `width` = 0 → canvas не изменился, в console warning от validation.

## Критерии приёмки

- Все правки идут через `updateNode`; компоненты не мутируют документ напрямую.
- Невалидный документ никогда не попадает в store (подтверждено тестом).
- Пять шагов визуальной проверки воспроизводятся.
- Обязательные команды зелёные.
